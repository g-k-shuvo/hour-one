## Workflow

1. **Create Feature**: Run `/feature-new <slug>` to create a new feature spec
2. **Plan**: Run `/plan <slug>` to plan the implementation
3. **Execute**: Run `/execute <slug>` to implement the feature
4. **Test**: Run `/tests <slug>` to add/run tests
5. **PR**: Run `/pr <slug>` to create a pull request
6. **Review**: Run `/pr-comments <slug>` to address review feedback
7. **Submit**: Run `/submit <slug>` to finalize and merge

## Template

See the feature specification template in the organization standards or create a new feature using `/feature-new`.
